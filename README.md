# BARTOK

Peer-to-peer marketplace for **verifiable AI inference**, the "Uber of LLMs" — live at
**[bartok-ten.vercel.app](https://bartok-ten.vercel.app)**. Buyers ("Ritas") chat and pay
in **BarŦoks** (Ŧ; on-chain token, decimals = 2, pegged 1 base unit = 1 Basic LLM token)
from a wallet that is silently a **Guardian multisig**; the seller (João) serves each
reply from real models; every reply is **cryptographically verified** (which model ran,
for how many tokens) without exposing the API key, prompt, or answer; and sessions settle
**privately on Miden testnet** through **Bartok-Guardian**, an application Guardian that
must co-sign every value-moving transaction. The buyer UX hides all of it — no wallet,
note, escrow, or chain vocabulary ever reaches Rita.

## What's here

See **`ARCHITECTURE.md`** for how the parts compose, and **`DEMO.md`** for the full runbook.

- **`ux-prototype/`**: Rita's app (Vite + vanilla JS + `@miden-sdk/miden-sdk` 0.15; the
  Guardian multisig client is vendored under `vendor/`), João's live seller dashboard
  (`seller.html`, SSE-fed), and the zero-dep bridge (`server.js`: sessions, tier routing,
  zkTLS pipeline, Guardian-countersigned settlement, credits funnel, rate limits).
- **`guardian/`**: runner + keys for **Bartok-Guardian** (dedicated fork at
  `guelowrd/bartok-guardian`, branch `bartok`; state and keys are gitignored).
- **`zktls-spike/`**: the proof lane. TLSNotary MPC-TLS over a real `openrouter.ai` call,
  selective disclosure of the labeled `model` + `usage` fields, a real notary keypair
  (gitignored, generated by `openrouter_keygen`), and the **oracle** that verifies proof +
  notary and attests `charge = tokens × tier price` in base units.
- **`miden/`**: the on-chain half on protocol **v0.15**, standalone workspace.
  `contracts/settlement-note/` is the BartokSettlement note (escrow in; charge as the note
  arg at consumption; two P2ID notes out), plus the testnet bins (`setup_accounts`,
  `setup_multisigs`, `new_faucet`, `fund_buyer`, `build_escrow`, `settle_session`,
  `joao_sweep`, `smoke_escrow`) and the MockChain regression test.

## Run it

Prereqs once (details in `DEMO.md`): `zktls-spike/setup.sh`, an OpenRouter key in
`zktls-spike/.env`, `openrouter_keygen`, `setup_accounts` + `setup_multisigs`,
`npm install` in `ux-prototype/`.

```bash
./serve.sh                         # Bartok-Guardian + the bridge (idempotent, keeps Mac awake)
cd ux-prototype && npm run dev     # Rita's app -> http://localhost:5173
# seller dashboard                 -> http://localhost:8787/seller.html
```

Public testing: `./tunnel.sh` (ngrok static domain) + the Vercel deployment; see `DEMO.md`.

## Headless checks

```bash
./stitch.sh --mock   # MockChain settlement regression (fast, offline)
./stitch.sh          # cycle-1 E2E of the proof lane + a plain-wallet settlement
```
